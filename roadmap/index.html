v1 Architecture (Current Reality)

Concise: what exists on-chain today and how funds flow.

Tax Flow

  1. Buys/sells are taxed in MMM (bps configurable)
  2. Taxes accumulate in TaxVault
  3. RewardVault can sweep MMM out (RewardVault-only permission)
Implemented

Reward Flow

  1. Reward distribution is snapshot-style accounting
  2. Claims are gated by minHoldTimeSec, claimCooldown, and minBalance
  3. Eligible supply is computed using the locked exclusion rule
Implemented Live-tested
What v1 does NOT do
  • No NFT boosts (phase 2)
  • No advanced tiering / dynamic behavioral taxes (optional roadmap)
  • No “magic” liquidity automation — pair creation + LP seeding are explicit steps

Mainnet Launch Runbook (v1)

One step at a time; no guessing; everything is manifest-driven and reproducible.

0) Pre-flight environment

Required
  • Confirm deployer has enough MON for all transactions + safety margin
  • Confirm RPC for monadMainnet is set and stable
  • Confirm Uniswap V2 factory/router/wmon constants (above)
  • Confirm you are on the correct git commit you intend to ship

1) Create the pair (MMM / WMON)

Blocking
  • After MMMToken is deployed on mainnet, call Factory.createPair(MMMToken, WMON)
  • Persist the resulting pair address in the manifest (PAIR_ADDR)
  • Confirm the pair has code and matches factory expectations
Why this matters

v1 tax recognition relies on the pair being set. Mainnet deployment must lock the correct pair address.

2) Deploy MMM v1 (locked)

Required
  • Run locked deploy script (mainnet)
  • Script writes deployments/monadMainnet/latest.json
  • Router must be non-zero on mainnet

3) Manifest verification gate

Required
  1. Run preflight-from-manifest (must PASS)
  2. Run assert-manifest (must PASS)
  3. Run stamp-manifest (pins bytecode hashes + git commit)
  4. Re-run assert-manifest (warnings should be gone)

4) Add liquidity + verify trading path

Required
  • Seed LP with MMM + MON/WMON using Router02
  • Verify buy/sell route works (small trades)
  • Verify TaxVault receives MMM taxes and wiring remains correct

5) Ownership + operational hardening

Recommended
  • Transfer ownership to multisig (or timelock) for protocol contracts
  • Lock LP (or clearly publish LP policy)
  • Publish the stamped manifest (addresses + bytecode hashes)
  • Prepare monitoring: tax inflows, RewardVault params, claim anomalies
Go/No-Go criteria
  • Mainnet deploy script completes without manual patching
  • Manifest: stamped (gitCommit + bytecode hashes) and committed
  • Pair: created and verified (MMM/WMON) + liquidity seeded
  • preflight-from-manifest PASS on mainnet
  • assert-manifest PASS on mainnet (strict, router code present)

Post-Launch Roadmap

  • Docs: runbooks, incident procedures, manifest publishing
